# GST Safe Docs Repository

This repository contains planning and product documentation for the GST Safe MVP.

## Documents
- `docs/MVP_DEVELOPMENT_PLAN.md` — phased 30/60/90-day implementation plan
- `docs/PRODUCT_REQUIREMENTS_DOCUMENT.md` — MVP PRD, goals, requirements, acceptance criteria, metrics
- `docs/IMPLEMENTATION_BACKLOG.md` — epics, stories, acceptance criteria, sprint mapping
- `docs/TECHNICAL_SPEC_MVP.md` — service boundaries, APIs, schemas, matching rules, security, SLOs
- `docs/PILOT_EXECUTION_RUNBOOK.md` — pilot onboarding, KPI cadence, escalation, rollout gates

## How to use this repo
1. Start with PRD for scope alignment
2. Use backlog doc for sprint planning
3. Use technical spec for architecture and API implementation
4. Use pilot runbook for customer onboarding and validation
